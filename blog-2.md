the four pillars of OOP help organize code in a clean and scalable way.

Inheritance allows you to reuse code by creating a parent class and extending it in child classes. This reduces duplication and keeps shared logic in one place.

Polymorphism lets different classes use the same method name in different ways. This makes your code flexible because one interface can work with multiple implementations.

Abstraction hides complex implementation details and shows only the essential features. This makes the system easier to understand and use without worrying about internal logic.

Encapsulation keeps data and methods inside a class and restricts direct access. This protects data from unwanted changes and improves security and control.
